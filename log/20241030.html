Recently I've made the decision of moving away from Dunst to herbe. It's a smaller (~200 loc) notification tool that doesn't require dbus and is very easily scriptable. A GIF preview is available on the repository, if you want to see its simplicity and functionality yourself.

It's configured via a config.h file at make (suckless style). You don't need to run any daemons since herbe spawns a new client for every notification.

It has its limitations, the biggest one being it can only display one notification at a time. However, that doesn't take away from its usability and minimalism.

Herbe is also very easy to patch and write patches for because of its small codebase, but I didn't patch mine so I don't have anything to show off here.

What I did want to share with you is some ideas you could use if you want to customize herbe yourself:

  • A makeshift bar that displays system info like battery life and the current time
  • Paired with a cronjob and a file, a to-do list and with the dynamic patch, also refresh the todo list when you edit the file
  • A context menu, for example one that asks you whether you want to poweroff or restart

... and more. Overall, I think herbe is a great tool, and even though it doesn't have all of the functionality that dunst has, it's still very neat for such a small project.
